Two-node wireless timing

Sprint timing,
without the stopwatch.

Accelence is a wireless lap/sprint timing system built on Arduino Nano Every boards, HC-SR04 ultrasonic sensors, and HC-05 Bluetooth — paired with a live web dashboard that shows every lap the moment it happens.

The Problem

One stopwatch can't cover two lines.

Manually timing sprints or laps with stopwatches is imprecise and doesn't scale — one person can't accurately time multiple runners crossing multiple points. Accelence automates this with two unmanned sensor gates that communicate wirelessly and log every lap automatically.

How It Works

Two gates, one wireless handoff.

Each gate is an unmanned Arduino node with an ultrasonic sensor. They talk to each other over Bluetooth so nothing needs to be wired across the track.

1

Sender gate detects the start

An Arduino with an ultrasonic sensor watches the start line. The moment it detects a runner crossing, it sends a signal wirelessly over Bluetooth to the receiver.

2

Receiver gate closes the lap

A second Arduino at the finish line receives that signal and starts an internal timer. Its own ultrasonic sensor watches for the crossing that completes the lap.

3

Live dashboard shows the result

A lightweight local web app reads the receiver's data in real time and displays it on a clean, live-updating page — viewable from any phone or laptop on the same network, no app install required.

Hardware

Each gate, one enclosure.

The 3D model above is the gate enclosure — an Arduino Nano Every, HC-SR04 sensor, and HC-05 Bluetooth module packed into a single unmanned unit. Two of these, one per line, are all the hardware Accelence needs on the track.

  • Arduino Nano Every — reads sensors, runs the timing logic
  • HC-SR04 ultrasonic sensor — detects the runner crossing
  • HC-05 Bluetooth module — links the two gates with no cable

Key Features

      
Built to run itself.

    

    

      

        
        
        
        
Fully wireless between gates

        
No cable connecting the two sensor stations, removing the distance limitations of a wired setup.

      

      

        
        
        
        
Noise-resistant detection

        
Debounced sensor readings and a tuned scan rate prevent false triggers from sensor echo/ringing.

      

      

        
        
        
        
Bluetooth link health-check

        
Both boards continuously verify the connection is alive, not just assumed.

      

      

        
        
        
        
Live, shareable results

        
Anyone on the local network can pull up the dashboard on their own device during a demo or event.

      

      

        
        
        
        
No internet dependency

        
Runs entirely on a local network — including a personal hotspot — making it viable anywhere, including venues with unreliable WiFi.

      

      

        
        
        
        
<1.5s debounce window

        
Cooldown timing tuned so a runner's second footfall never double-fires the same gate.
